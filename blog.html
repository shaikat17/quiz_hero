  • Question 2: Difference among localStorage, sessionStorage and cookies?
  • Ans: Local storage Local storage is the most recent mechanism. It allows for larger amounts of data to be stored, but the data is not deleted when the browser is closed. Local storage is useful for storing data that the user will need to access later, such as offline data. Session storage Session storage is similar to cookies, but the data is only stored for the current session. This means that the data will be deleted when the user closes the browser. Session storage is useful for storing data that is sensitive, such as login credentials. Cookies 1. Cookies are the oldest and most well-known mechanism. They are simple to use and well supported by browsers. However, they are limited to 4KB of data and are often used to store data that is not sensitive, such as user preferences.
  • Question 3: What is event loop? How does it work?
  • Ans: Event Loop event loop is a mechanism that allows the code to run asynchronously by processing events and callbacks. The event loop is a critical part of the JavaScript runtime environment and is responsible for handling user input, network requests, and other asynchronous tasks. It's working process The event loop checks the call stack and event queue, executes any synchronous code, and adds asynchronous code to the event queue. When the call stack is empty, the event loop takes the first task from the event queue and adds it to the call stack, which will be executed immediately. The process repeats until the call stack and event queue are empty. The event loop allows JavaScript to handle asynchronous tasks without blocking the main thread, making web applications more responsive and efficient.
